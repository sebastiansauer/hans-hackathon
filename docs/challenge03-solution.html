1 Daten verstehen

1.1 Aufgaben

Datensatz verschlanken

Aufbauend auf dem Ergebnis der letzten Challenge:

  1. Welche Spalten halten Sie für nicht relevant, um das Nutzerverhalten besser zu verstehen?. Begründen Sie die Auswahl.
  2. Entfernen Sie redundante Spalten (nach eigenem Ermessen und mit Begründung).
  3. Behalten Sie - neben allgemeinen Informationen zum Nutzer - nur die Spalten mit subtitle im Namen.
  4. Reparieren Sie die Spalten mit Datumsangaben.
  5. Entfernen Sie Spalten, die Bilddaten enthalten.
  6. Entfernen Sie Zeilen, die Daten von Admins, Developern oder lecturers enthalten.

1.2.3 2

Begrenzen wir uns hier daher erstmal auf subtitle und Zeitangaben timestamp. Die übrigen Spalten entfernen wir fürs Erste (s. der folgende Abschnitt).

1.2.4 3

d_only_subtitles_and_timestamps <-
  d |> select(idVisit, contains("subtitle"), contains("timestamp"))

Schon deutlich (etwas) übersichtlicher.

d_students_only <-
d_time_repaired |> 
  filter(!str_detect(actionDetails_0_subtitle, "developer|lecturer|admin"))

Der vertikale Balken | steht für das logische “Oder” in Regex.
